Modelos Clasicos

Modelo en cascada: Se define así por la posición de las fases, este modelo no permite el inicio de una fase hasta que no esté completamente terminada la fase anterior.

Modelo incremental: En este modelo se utilizan elementos del modelo en cascada, pero de una forma interactiva, el foco de este modelo es entregar un producto funcional en cada fase incremental; por ende, es de gran utilidad cuando para el proyecto no se cuenta con el equipo necesario para cumplir de una vez por completo la entrega de este.

Modelo en prototipos: Este modelo es construido en poco tiempo y usando las herramientas para hacer del prototipo más fácil y entendible.

Modelo en espiral: Las actividades se conforman en un espiral en la que cada bucle o interacción representa un conjunto de actividades, no están fijadas a prioridad, sino que las siguientes se eligen en función de análisis de riesgo comenzando por el bucle interior.
